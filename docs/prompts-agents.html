☞ ¿Cómo funciona?

Los prompts de este proyecto son plantillas de personalidad para agentes de IA. Cuando invocas un prompt, el agente se caracteriza según la información que le proporcionas: adopta un rol, analiza la codebase desde esa perspectiva y guía una sesión interactiva contigo.

El truco: puedes agregar lore y datos extra al final del prompt para que el agente se especialice aún más. Sin datos extra, actúa como codebase owner general. Con datos extra (ej. contexto de un componente, un artículo, un rol concreto), el agente se adapta y acota sus respuestas a esa información.

Uso en VS Code / GitHub Copilot:
.github/prompts/*.prompt.md → invócalos desde el chat con @workspace /hacker-devops o referencia el archivo y pega contexto adicional al final.

Uso con cualquier agente (Claude, ChatGPT, etc.):
Copia el contenido del .prompt.md, pégalo como system prompt o primer mensaje, y añade debajo tu lore/datos extra.

🔧 hacker-devops

.github/prompts/hacker-devops.prompt.md
Inicia una sesión interactiva asumiendo el rol de un hacker DevOps experimentado con actitud asertiva, crítica pero constructiva, que acaba de descubrir esta codebase. Analiza las puertas de entrada del proyecto (README + GH Pages), cruza documentación con realidad (git log, barrel, backlog) y guía la mejora paso a paso.
⚡ Clave: el prompt acepta información extra al final. Sin ella, el agente procede como codebase owner general. Con lore adicional (artículos, contexto de un componente, rol específico, datos técnicos), el agente se especializa y acota sus análisis a esa información.

Fases de la sesión

  1. La Revisión Inquisitiva — Analiza la puerta de entrada (README, GH Pages) y cruza con barrel + backlog
  2. Adaptación y Refactorización (DRY) — El usuario asigna un rol; rescaneo desde esa perspectiva
  3. Definición del Plan de Ataque — Localiza tareas en BACKLOG.md y propone iniciativa concreta
  4. El Offboarding / Salida Limpia — Valida que la codebase explica cómo probar, compilar y salir
  5. El Cierre Definitivo — Aplica mejoras acordadas, actualiza backlog, cierra sesión

Fuentes de verdad que consulta

BACKLOG.md specs/ git log src/index.ts examples/

☞ Cómo Crear tu Propio Prompt

1. CREA EL ARCHIVO
touch .github/prompts/mi-agente.prompt.md
2. AÑADE EL FRONTMATTER
# .github/prompts/mi-agente.prompt.md --- name: mi-agente description: "Descripción corta del rol." --- Tu prompt aquí. Define la personalidad, las fuentes de verdad que debe consultar, y las fases de la sesión.
3. USA EL PROMPT
# En VS Code con Copilot Chat @workspace /mi-agente # O cópialo y pégalo en cualquier agente # añadiendo lore al final para especializar
Tip: Reutiliza la estructura de fases de hacker-devops como base y personaliza las fuentes de verdad según tu área de interés.
